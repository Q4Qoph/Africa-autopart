# Contributing to Africa Auto Parts

Thank you for your interest in contributing!

## Workflow

1. Fork the repository and create your branch from `main`.
2. Branch naming: `feat/<short-description>`, `fix/<short-description>`, `chore/<short-description>`.
3. Make your changes and write or update tests where applicable.
4. Ensure all linting passes: `npm run lint` (inside `src/frontend`).
5. Open a pull request using the PR template.

## Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add supplier search filter
fix: correct order status mapping
chore: update dependencies
docs: add architecture diagram
```

## Code Style

- TypeScript strict mode is enabled — no `any` unless justified.
- Prefer named exports over default exports.
- Keep components small and single-responsibility.

## Reporting Issues

Use the GitHub Issue templates for bug reports and feature requests.
